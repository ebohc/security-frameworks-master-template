A single reference workbook mapping 9 major security frameworks 
side by side — from two perspectives: the analyst who implements 
controls and the auditor who certifies them.

## Frameworks covered

| Framework | What it covers |
|-----------|---------------|
| SOC 2 Type II | Trust Service Criteria — Security, Availability, Confidentiality |
| ISO 27001:2022 | International ISMS standard — Annex A controls |
| NIST CSF v1.1 | Govern, Identify, Protect, Detect, Respond, Recover |
| NIST SP 800-53 Rev 5 | Security and privacy controls for federal systems |
| NIST RMF | Prepare, Categorize, Select, Implement, Assess, Authorize, Monitor |
| PCI DSS v4.0 | Payment card data security — 12 requirements |
| HIPAA Security Rule | Administrative, physical, and technical safeguards for ePHI |
| CIS Controls v8 | 18 prioritised security controls — IG1/IG2/IG3 |
| ITGC | Access, change management, operations, and system development |

## What's in each framework tab

Every framework tab contains:

- **Requirement ID** — the official control reference
- **Control domain** — the area of security being addressed
- **Requirement** — plain-English description of what is required
- **Analyst — What to implement** — what to build, configure, and deploy
- **Analyst — Evidence to maintain** — what documentation to keep
- **Auditor — Evidence requested** — what an auditor will ask for
- **Auditor — What a PASS looks like** — the exact standard required to pass
- **Auditor — Common failures** — the most frequent reasons controls fail
- **Review frequency** — how often the control must be reviewed
- **RAG status** — Green / Amber / Red dropdown with conditional formatting

---

## Additional tabs

**Framework Map** — 30 control domains mapped across all 9 frameworks 
simultaneously. Shows where frameworks overlap so you can satisfy 
multiple requirements with a single control.

**Glossary** — 51 terms defined in plain English. Every acronym used 
in the workbook is explained here.
